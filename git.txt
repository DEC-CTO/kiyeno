 📚 Git 버전 관리 활용법

  1. 이전 커밋 내용 확인

  # 커밋 히스토리 보기
  git log --oneline    # 간단한 한 줄씩
  git log --graph      # 시각적 브랜치 구조

  # 특정 파일의 변경 히스토리
  git log --follow public/js/app-services.js

  # 특정 커밋의 내용 확인
  git show [커밋해시]

  2. 코드 비교 및 분석

  # 현재와 이전 버전 비교
  git diff HEAD~1 public/js/app-services.js

  # 특정 커밋들 간 비교
  git diff [이전커밋] [현재커밋] --name-only

  # 변경된 줄 수 확인
  git diff --stat HEAD~5

  3. 안전한 코드 복원

  # 특정 파일만 이전 버전으로 복원
  git checkout [커밋해시] -- public/js/app-services.js

  # 전체 프로젝트를 특정 시점으로 되돌리기
  git reset --hard [커밋해시]

  # 임시로 이전 상태 확인 (읽기 전용)
  git checkout [커밋해시]

  🚀 오늘 상황에서 Git 활용 예시

  문제 발생 시 Git 워크플로우:

  1. 문제 발생 확인
     → git status (현재 변경사항 확인)

  2. 언제부터 문제였는지 파악
     → git log --oneline -10 (최근 10개 커밋 확인)

  3. 정상 작동했던 버전 찾기
     → git show [커밋해시] (해당 시점 코드 확인)

  4. 빠른 복원
     → git checkout [정상커밋] -- public/js/app-services.js

  💡 Git의 엄청난 장점들

  1. 정확한 시점 복원

  - 👍 "어제 오후 3시 버전"으로 정확히 돌아가기 가능
  - 👍 특정 기능 추가 전 상태로 되돌리기
  - 👍 "마지막 정상 작동" 버전 즉시 찾기

  2. 부분 복원 가능

  - 👍 문제된 파일만 복원, 다른 파일은 그대로 유지
  - 👍 특정 함수나 기능만 이전 버전으로 되돌리기
  - 👍 실험적 변경사항만 취소

  3. 변경 히스토리 추적

  - 👍 "누가, 언제, 무엇을, 왜" 변경했는지 완전 추적
  - 👍 버그 도입 시점 정확히 파악
  - 👍 코드 진화 과정 완전 기록

  4. 안전한 실험

  - 👍 브랜치로 실험 → 문제 시 즉시 원복
  - 👍 여러 버전 동시 관리 및 비교
  - 👍 되돌리기도 되돌릴 수 있음 (Git은 거의 모든 것을 기억)

  🎯 결론

  Git = 개발자의 "타임머신"

  오늘 같은 상황이라면:
  - ❌ Git 없었다면: 수동 백업 파일들 뒤져가며 복원 (몇 시간)
  - ✅ Git 있다면: git checkout HEAD~2 -- app-services.js (10초)

  장기적 이점:
  - 🔸 모든 변경사항이 기록되어 문제 원인 추적 용이
  - 🔸 안전한 실험과 빠른 복원으로 개발 속도 향상
  - 🔸 "실수 두려움" 없이 대담한 리팩토링 가능
  - 🔸 팀 협업 시 충돌 해결과 변경 추적 자동화
